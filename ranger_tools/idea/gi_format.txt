В файле записано:
- один заголовок файла
- несколько заголовков слоев (их количество записано в заголовке файла)
- данные всех слоев (их размеры и смещения записаны в заголовке соответствующего слоя)

Порядок хранения байтов - little-endian


GI header - 64 байта:
0x00: signature = 0x00006967 = b'gi\0\0' - сигнатура файла
0x04: version = 1 - версия (всегда 1?)
0x08: start_X - границы изображения
0x0c: start_Y
0x10: finish_X
0x14: finish_Y
0x18: r_bitmask - битовые маски цветов (вероятно используются только для различения двух подтипов 0 типа слоя)
0x1c: g_bitmask
0x20: b_bitmask
0x24: a_bitmask
0x28: frame_type - тип слоя (0, 1, 2, 3, 4, 5)
0x2c: layer_count - количество слоев
0x30: unknown1 - countUpdateRect
0x34: unknown2 - smeUpdateRect
0x38: unknown3 - rz0
0x3c: unknown4 - rz1

Layer header - 32 байта:
0x00: seek - смещение данных слоя
0x04: size - размер данных слоя
0x08: start_X - границы слоя
0x0c: start_Y
0x10: finish_X
0x14: finish_Y
0x18: unknown1 - r0
0x1c: unknown2 - r1



Frame type 0: 
    Один слой:
        RGB16 или RGBA32 в зависимости от маски
Frame type 1: 
    Один слой:
        RGB16 оптимизированный (пустые пиксели пропущены)
Frame type 2: 
    Три слоя: 
        RGB16 - непрозрачные пиксели
        RGB16 - полупрозрачные пиксели
        A6 - альфа для полупрозрачных пикселей. 
    Слои оптимизированы (пустые пиксели пропущены)
Frame type 3: 
    Два слоя:
        indexed RGB
        indexed A
Frame type 4: 
    Один слой:
        indexed RGBA
Frame type 5: 
    delta frame


Layer data (frame type 0):
    Формат хранения зависит от маски:

    r_bitmask = 0x00FF0000
    g_bitmask = 0x0000FF00
    b_bitmask = 0x000000FF
    a_bitmask = 0xFF000000
        Формат: 32-битный с альфой, RGBA32
        Порядок хранения цветов: b, g, r, a

    r_bitmask = 0xF800
    g_bitmask = 0x07E0
    b_bitmask = 0x001F
    a_bitmask = 0x0000
        Формат: 16-битный без альфы, RGB16
        rgb16

    В данных слоя содержатся все пиксели, записанные по строкам




Layer data (frame type 2):
    0x00: size - размер данных (на 16 меньше, чем в заголовке слоя)
    0x04: width - ширина изображения
    0x08: height - высота изображения
    0x0c: unknown
    0x10: данные слоя (size байт)


    Данные слоя состоят из блоков. Каждый блок в начале содержит управляющий байт, после которого записано несколько пикселей
    Значения управляющего байта (=byte):
        0x00, 0x80
            нужно пропустить считывание текущей строки, перейти к следующей
            блок не содержит пикселей
        0x01..0x7f
            нужно пропустить считывание (byte) пикселей
            блок не содержит пикселей
        0x81..0xff
            блок содержит (byte-0x80) пикселей
